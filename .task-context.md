# DevOps Architect Task Context

## Mission
Fix database schema initialization failure and automate migration execution in deployment pipeline.

## Project Details
- **Application**: MGA SOAP Calculator
- **Server**: grimm-lin (Fedora 42)
- **Container Runtime**: Rootless Podman (user: grimm)
- **Database**: PostgreSQL 17.2 in mga-postgres container
- **API**: FastAPI in soap-calculator-app container
- **Current Issue**: Database running but schema uninitialized (no tables)

## Root Cause
Alembic migrations never executed during deployment. API container started but no migration step in deployment workflow.

## Critical Constraints
- **MGA Confidential**: All test data stays on grimm-lin server
- **Rootless Podman**: Must use XDG_RUNTIME_DIR for user grimm
- **Ansible Scope**: user-level services only (--user flag)
- **Idempotency**: All changes must be safely re-runnable

## Infrastructure Context
- Container orchestration via systemd user services
- PostgreSQL: port 5432 (internal), persistent volume mga-postgres-data
- API: port 8000, linked to PostgreSQL container
- Deployment via Ansible playbooks from local machine

## Expected Deliverables
1. Migration execution results
2. Schema verification output
3. Updated Ansible playbook with exact line numbers
4. Endpoint test results
5. Deployment gap analysis
6. Agent response file with complete documentation
