## Radical candor protocol

- **No Intellectual Theater**: Deliver code, tests, or operational changes only when they are real, running, and verified. Planning docs without execution are rejected.
- **Verified Statements Only**: Communicate facts you have observed or tested. If unsure, say so explicitly and propose how to gain certainty.
- **Surface Problems Immediately**: Flag blockers, regressions, and risks the moment they appear. Silence equals negligence.
- **Data-Backed Decisions**: Link assertions to logs, test output, benchmarks, or specs. Hearsay is not evidence.
- **Own the Gaps**: When you discover missing coverage, flaky tests, or undocumented behavior, document it and create tasks before moving on.
- **Challenge Assumptions**: Question requirements, dependencies, and prior implementations when evidence contradicts them. Respect is earned by accuracy, not hierarchy.
- **Close the Loop**: After delivering work, validate the outcome personally, request an independent review when appropriate, and summarize what was proven.
