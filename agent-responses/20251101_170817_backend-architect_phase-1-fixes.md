## Issues Fixed

### 1. Password Security (CRITICAL) ✅

**Issue:** User model accepted plaintext passwords - no validation that `password_hash` was actually bcrypt-hashed

**Fix Applied:**
- Added `@validates('hashed_password')` decorator to User model
- Validator checks for bcrypt hash format ($2a$, $2b$, or $2y$ prefix)
- Validator enforces minimum hash length (59 chars)
- Raises `ValueError` with clear message if plaintext password attempted
- Added comprehensive docstring explaining security requirement

**File Modified:** `app/models/user.py`

**Code Added:**
```python
@validates('hashed_password')
def validate_hashed_password(self, key, value):
    """Validate that password is properly bcrypt-hashed

    Prevents accidental plaintext password storage by verifying
    bcrypt hash format ($2a$, $2b$, or $2y$ prefix with proper structure).
    """
    if value is None:
        return value

    if not value.startswith(('$2a$', '$2b$', '$2y$')):
        raise ValueError(
            "Password must be bcrypt-hashed before assignment. "
            "Use passlib.hash.bcrypt.hash('your_password') to hash passwords."
        )

    if len(value) < 59:
        raise ValueError(
            f"Invalid bcrypt hash format - too short (got {len(value)} chars, expected ≥59)."
        )

    return value
```

### 2. Timestamp Inconsistency (HIGH) ✅

**Issue:** Mixed client/server timestamp generation created timezone bugs
- Models used `default=datetime.utcnow` (client-side)
- Migration used `server_default=sa.func.now()` (server-side)
- This mismatch could cause subtle timezone issues in production

**Fix Applied:**
- Removed all `default=datetime.utcnow` from models
- Standardized on `server_default=sa.func.now()` for all timestamps
- PostgreSQL generates all timestamps server-side in UTC
- Ensures consistent timezone handling across all operations

**Files Modified:**
- `app/models/user.py` (created_at, updated_at)
- `app/models/oil.py` (created_at, updated_at)
- `app/models/additive.py` (created_at, updated_at)
- `app/models/calculation.py` (created_at)

**Before:**
```python
created_at: Mapped[datetime] = mapped_column(
    DateTime(timezone=True),
    default=datetime.utcnow,  # ❌ Client-side
    nullable=False,
)
```

**After:**
```python
created_at: Mapped[datetime] = mapped_column(
    DateTime(timezone=True),
    nullable=False,
    server_default=sa.func.now(),  # ✅ Server-side
    comment="UTC timestamp generated by PostgreSQL server",
)
```

### 3. Missing Database Triggers (HIGH) ✅

**Issue:** `updated_at` columns won't auto-update because Alembic ignores SQLAlchemy's `onupdate` parameter

**Fix Applied:**
- Created new migration file: `migrations/versions/002_add_triggers_and_constraints.py`
- Added PostgreSQL trigger function `update_updated_at_column()`
- Applied trigger to all tables with `updated_at` columns:
  - users
  - oils
  - additives

**Migration Added:**
```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_users_updated_at
    BEFORE UPDATE ON users
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();

-- Similar triggers for oils and additives tables
```

**File Created:** `migrations/versions/002_add_triggers_and_constraints.py`

@pytest.mark.unit
async def test_user_model_creation(test_db_session: AsyncSession):
    """Test user model with UUID primary key and bcrypt password hashing"""
    # Hash password using bcrypt
    plaintext_password = "test_password_123"
    hashed_password = bcrypt.hash(plaintext_password)

    user = User(email="test@example.com", hashed_password=hashed_password)

    test_db_session.add(user)
    await test_db_session.commit()
    await test_db_session.refresh(user)

    # Verify it's a bcrypt hash
    assert user.hashed_password.startswith(('$2a$', '$2b$', '$2y$'))
    # Verify password can be verified
    assert bcrypt.verify(plaintext_password, user.hashed_password)
    # Verify wrong password fails
    assert not bcrypt.verify("wrong_password", user.hashed_password)
